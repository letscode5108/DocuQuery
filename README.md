# 📄 DocuQuery — AI-Powered Document Q&A API

A FastAPI backend that lets users upload PDFs, audio, and video files and ask questions about them using RAG (Retrieval-Augmented Generation). Documents are embedded with **Gemini** and stored in **Pinecone**; answers are generated by **Groq LLaMA 3.3 70B**. Audio/video files are transcribed with **Groq Whisper** and support timestamp-aware Q&A.

---

## ✨ Features

- **PDF upload & Q&A** — extract text, chunk, embed, and query with semantic search
- **Audio/Video upload** — transcribe with Whisper, embed segments with timestamps, seek to exact moments
- **Cross-document search** — query across all your uploaded files at once
- **Summaries** — generate concise summaries for any document or media file
- **Timestamp search** — find every moment a topic is discussed in a recording
- **JWT auth** — register/login, all routes are user-scoped
- **Cloudinary storage** — original files stored and served via CDN

---

## 🏗️ Architecture

```
Client
  │
  ├── POST /register / /token          (auth)
  │
  ├── POST /documents/                 (PDF upload → Cloudinary + Pinecone)
  ├── POST /media/                     (audio/video → Whisper → Pinecone)
  │
  ├── POST /query/                     (Q&A on a single PDF)
  ├── POST /query-media/               (Q&A on audio/video with timestamps)
  ├── POST /query-all/                 (cross-document search)
  │
  ├── POST /summarize/                 (summarize any document)
  └── POST /timestamps/               (find topic timestamps in media)
```

**Stack:**

| Layer | Technology |
|---|---|
| API | FastAPI |
| Auth | JWT (python-jose) + bcrypt |
| Embeddings | Gemini `gemini-embedding-001` (3072-dim) |
| LLM | Groq `llama-3.3-70b-versatile` |
| Transcription | Groq `whisper-large-v3` |
| Vector DB | Pinecone (serverless, cosine) |
| File storage | Cloudinary |
| PDF parsing | PyMuPDF (fitz) |
| Video → Audio | ffmpeg |
| Database | SQLAlchemy (any SQL backend) |

## ⚙️ Setup

### Prerequisites

- Python 3.10+
- `ffmpeg` installed and on your `PATH` (required for video uploads)
- Accounts/API keys for: Groq, Google AI (Gemini), Pinecone, Cloudinary

### 1. Clone & install

```bash
git clone https://github.com/your-org/documind.git
cd documind
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Configure environment

Copy the example env file and fill in your keys:

```bash
cp .env.example .env
```

`.env` contents:

```env
# Auth
SECRET_KEY=your-secret-key-min-32-chars

# AI Services
GROQ_API_KEY=gsk_...
GOOGLE_API_KEY=AIza...

# Pinecone
PINECONE_API_KEY=pcsk_...

# Cloudinary
CLOUDINARY_CLOUD_NAME=your-cloud-name
CLOUDINARY_API_KEY=your-api-key
CLOUDINARY_API_SECRET=your-api-secret

# Database
DATABASE_URL=sqlite:///./app.db    # or postgresql://user:pass@host/db
```

### 3. Initialize the database

```bash
python -c "from database import Base, engine; import models; Base.metadata.create_all(engine)"
```

### 4. Run

```bash
uvicorn main:app --reload --port 8000
```

Interactive docs available at `http://localhost:8000/docs`.

## 🔐 Authentication

All document and query endpoints require a Bearer token.

### Register

```http
POST /register?email=user@example.com&password=secret
```

```json
{ "id": 1, "email": "user@example.com" }
```

### Login

```http
POST /token
Content-Type: application/x-www-form-urlencoded

username=user@example.com&password=secret
```

```json
{ "access_token": "eyJ...", "token_type": "bearer" }
```

Include the token on all subsequent requests:

```
Authorization: Bearer eyJ...
```

## 🔒 Notes

- The Pinecone index `pdf-documents` is created automatically on startup with dimension `3072` (Gemini embedding-001). If an existing index has the wrong dimension it will be deleted and recreated.
- Each document gets its own Pinecone namespace (`doc_{id}`), so queries are always isolated per document unless using `/query-all/`.
- Video files require `ffmpeg` on the server PATH; audio is extracted as mono 16kHz MP3 before transcription.
- JWT tokens expire after 24 hours.
